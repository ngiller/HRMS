## 2. API Design Standards

### 2.1 Base URL

```
Production:  https://api.hrms.company.com/api/v1
Staging:     https://staging-api.hrms.company.com/api/v1
Development: http://localhost:8080/api/v1
```

### 2.2 Standard Response Format

**Success Response:**

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "page": 1,
    "per_page": 25,
    "total": 150,
    "total_pages": 6
  }
}
```

**Error Response:**

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "NIK harus 16 digit angka",
    "field": "nik",
    "details": [
      { "field": "nik", "message": "NIK harus 16 digit angka" }
    ]
  },
  "request_id": "req_abc123"
}
```

### 2.3 HTTP Methods & Conventions

| Method | Usage | Description |
|--------|-------|-------------|
| `GET` | Read | List (paginated) or detail (`/:id`) |
| `POST` | Create | Create resource |
| `PUT` | Update (full) | Replace entire resource |
| `PATCH` | Update (partial) | Partial update |
| `DELETE` | Delete (soft) | Sets `deleted_at` |

### 2.4 Pagination

**Request:**

```
GET /api/v1/employees?page=1&per_page=25&sort=full_name&order=asc&search=budi
```

**Params:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | int | 1 | Page number |
| `per_page` | int | 25 | Items per page (max 100) |
| `sort` | string | `created_at` | Sort column |
| `order` | string | `desc` | `asc` or `desc` |
| `search` | string | — | Full-text search |

### 2.5 Standard Headers

| Header | Description |
|--------|-------------|
| `Authorization: Bearer <token>` | JWT access token |
| `X-Request-ID` | Request tracing (auto-generated) |
| `Content-Type: application/json` | Request body type |
| `Accept-Language: id` | Language preference (id/en) |

### 2.6 Approval Trail Format

Semua request dengan approval menggunakan format JSONB:

```json
{
  "approval_trail": [
    {
      "level": 1,
      "approver_id": "uuid",
      "approver_name": "Budi Hartono",
      "status": "approved",
      "note": "Setuju",
      "date": "2026-06-11T08:30:00Z"
    }
  ]
```

## 3. Authentication & Authorization

### 3.1 JWT Token Strategy

- **Access Token:** 15 menit expiry, disimpan di memory (JS variable)
- **Refresh Token:** 7 hari expiry, disimpan di httpOnly cookie
- **Algorithm:** HS256 (HMAC-SHA256)
- **Claims:**
  ```json
  {
    "sub": "employee_uuid",
    "email": "budi@company.com",
    "role": "employee",
    "permissions": { ... },
    "iat": 1718000000,
    "exp": 1718000900
  }
  ```

### 3.2 Token Refresh Flow

```javascript
// Frontend interceptor
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      const refreshToken = getCookie('refresh_token');
      const { access_token } = await auth.refresh(refreshToken);
      setAccessToken(access_token);
      error.config.headers.Authorization = `Bearer ${access_token}`;
      return api(error.config);
    }
    return Promise.reject(error);
  }
);
```

### 3.3 RBAC Enforcement

```go
// Middleware pattern
func RBAC(permission string, action string) fiber.Handler {
    return func(c *fiber.Ctx) error {
        claims := c.Locals("claims").(jwt.MapClaims)
        permissions := claims["permissions"].(map[string]interface{})
        
        modPerms := permissions[permission].(map[string]interface{})
        if !modPerms[action].(bool) {
            return c.Status(403).JSON(fiber.Map{
                "success": false,
                "error": fiber.Map{
                    "code": "FORBIDDEN",
                    "message": "Anda tidak memiliki akses ke modul ini",
                },
            })
        }
        return c.Next()
    }
}
// Usage: r.Get("/employees", RBAC("employee", "read"), handler.List)
```

### 3.4 Approval Line Resolution

Setiap karyawan memiliki `approval_line_id` (FK ke employees). Resolution:

```go

    // 4. Jika sampai puncak, return HR Manager
```

## 4. API Endpoints — Autentikasi

### 4.1 Login

```
POST /api/v1/auth/login
```

**Request Body:**

```json
{
  "email": "budi@company.com",
  "password": "P@ssw0rd123",
  "device_info": {
    "user_agent": "Mozilla/5.0 ...",
    "platform": "web",
    "app_version": "1.0.0"
  }
}
```

**Response (200):**

```json
{
  "access_token": "eyJhbGciOi...",
  "expires_in": 900,
  "user": {
    "id": "uuid",
    "full_name": "Budi Hartono",
    "email": "budi@company.com",
    "role": "employee",
    "photo_url": "https://storage.company.com/photos/budi.jpg",
    "permissions": { "employee": { "read": true }, ... }
  }
}
```

**Response (401):**

```json
{
  "error": { "code": "INVALID_CREDENTIALS", "message": "Email atau password salah" }
}
```

**Response (423 — Locked):**

```json
{
  "error": { "code": "ACCOUNT_LOCKED", "message": "Akun terkunci karena 5x gagal login. Coba lagi dalam 15 menit" }
}
```

**Business Rules:**
- Max 5 failed attempts → lockout 15 menit (catat di `login_attempts`)
- Jika akun `is_locked = true` dan `locked_until > NOW()` → tolak login
- Jika akun `deleted_at IS NOT NULL` → tolak login
- Hash bcrypt/argon2 untuk verifikasi password
- Generate access + refresh token, simpan refresh di `user_sessions`

### 4.2 Refresh Token

```
POST /api/v1/auth/refresh
```

**Cookie (httpOnly):** `refresh_token=xxx`

**Response (200):**

```json
{
  "access_token": "eyJhbGciOi..."
}
```

**Business Rules:**
- Refresh token single-use (setelah dipakai, hapus session lama, buat baru)
- Jika refresh token expired/invalid → user harus login ulang
- Rotate refresh token setiap kali digunakan

### 4.3 Logout

```
POST /api/v1/auth/logout
```

**Cookie:** refresh_token

**Response:** `{ "success": true }`

**Business Rules:**
- Hapus session dari `user_sessions`
- Hapus cookie refresh_token

### 4.4 Forgot Password

```
POST /api/v1/auth/forgot-password
```

**Request:**

```json
{
  "email": "budi@company.com"
}
```

**Response (200):** `{ "message": "Link reset password telah dikirim ke email Anda" }`

**Business Rules:**
- Generate token random 32-byte, simpan hash di `password_reset_tokens`
- Token expired dalam 60 menit
- Kirim email (via SendGrid/SMTP) dengan link: `https://app.company.com/reset-password?token=xxx`
- Jangan kasih tahu apakah email terdaftar (security best practice)

### 4.5 Reset Password

```
POST /api/v1/auth/reset-password
```

**Request:**

```json
{
  "token": "reset_token_here",
  "password": "NewP@ssw0rd456",
  "password_confirm": "NewP@ssw0rd456"
}
```

**Response (200):** `{ "message": "Password berhasil direset. Silakan login." }`

**Business Rules:**
- Validasi token dari `password_reset_tokens` (hash match + not expired + not used)
- Password: min 8 char, uppercase, lowercase, number, symbol
- Set `is_used = true` setelah sukses

```json
  "department_id": "uuid",

  "position_grade_id": "uuid",
  "approval_line_id": "uuid",
  "ptkp_status": "K1",
  "tax_method": "gross",
  "work_schedule_id": "uuid",

```

**Response (201):**

```json
{
  "id": "uuid",
  "message": "Karyawan berhasil dibuat"
}
```

**Validations:**
| Field | Rule |
|-------|------|
| `employee_id` | Required, unique, max 50 chars |
| `full_name` | Required, max 255 chars |
| `nik` | 16 digit numeric |
| `email` | Required, valid format, unique |
| `password` | Min 8 chars, uppercase, lowercase, number |
| `gender` | Required, enum `laki_laki` / `perempuan` |
| `phone` | Pattern: `+62` or `08` prefix, 10-15 digits |
| `join_date` | Required, not future date |

**Business Actions:**
1. Enkripsi `nik`, `npwp`, `bank_account`, `bank_name` dengan AES-256-GCM
2. Hash password dengan bcrypt (cost 12)
3. Buat `employee_salary_histories` entry dengan base_salary = 0
4. Buat `leave_balances` untuk tahun berjalan
5. Generate NIP otomatis (jika tidak diisi)
6. Catat di `activity_logs`

### 5.4 Update Karyawan

```
PUT /api/v1/employees/:id
```

**Request Body:** Sama seperti Create (semua field opsional — partial update via PATCH)

**Response (200):** `{ "id": "uuid", "message": "Data karyawan berhasil diupdate" }`

**Business Rules:**
- Catat perubahan di `employee_histories` jika field pekerjaan berubah
- Jika `approval_line_id` berubah, catat siapa yang mengubah
- Re-enkripsi data sensitif jika berubah


**Response (200):** `{ "message": "Karyawan dinonaktifkan" }`

**Business Rules:**

- Tidak bisa hapus jika memiliki request pending (cuti, reimbursement, dll)
- Catat `resigned_at` dan `resign_reason` jika resign

## 18. Security Implementation

### 18.1 AES-256-GCM Encryption (Go Layer)

```go
// internal/pkg/encrypt/aes.go
package encrypt

import (
    "crypto/aes"
    "crypto/cipher"
    "crypto/rand"
    "encoding/base64"
    "errors"
    "io"
)

var encryptionKey []byte

func Init(key string) {
    encryptionKey = []byte(key)
}

func Encrypt(plaintext string) (string, error) {
    block, err := aes.NewCipher(encryptionKey)
    if err != nil {
        return "", err
    }

    aead, err := cipher.NewGCM(block)
    if err != nil {
        return "", err
    }

    nonce := make([]byte, aead.NonceSize())
    if _, err := io.ReadFull(rand.Reader, nonce); err != nil {
        return "", err
    }

    ciphertext := aead.Seal(nonce, nonce, []byte(plaintext), nil)
    return base64.StdEncoding.EncodeToString(ciphertext), nil
}

func Decrypt(cipherBase64 string) (string, error) {
    ciphertext, err := base64.StdEncoding.DecodeString(cipherBase64)
    if err != nil {
        return "", err
    }

    block, err := aes.NewCipher(encryptionKey)
    if err != nil {
        return "", err
    }

    aead, err := cipher.NewGCM(block)
    if err != nil {
        return "", err
    }

    nonceSize := aead.NonceSize()
    if len(ciphertext) < nonceSize {
        return "", errors.New("ciphertext too short")
    }

    nonce, ciphertext := ciphertext[:nonceSize], ciphertext[nonceSize:]
    plaintext, err := aead.Open(nil, nonce, ciphertext, nil)
    if err != nil {
        return "", err
    }

    return string(plaintext), nil
}
```

### 19.2 SQL Injection Prevention

**✅ Aman — menggunakan sqlc untuk semua query database.**

sqlc menghasilkan **parameterized queries (prepared statements)** dari SQL murni, sehingga input user **tidak pernah** digabung langsung ke string SQL.

```sql
-- ❌ TIDAK BOLEH — string concatenation
SELECT * FROM employees WHERE email = '" + userInput + "'

-- ✅ Aman — sqlc parameterized query
-- name: GetEmployeeByEmail :one
SELECT * FROM employees WHERE email = $1 AND deleted_at IS NULL;
```

**Aturan Ketat:**
1. Semua query database HARUS melalui sqlc — **dilarang keras** menggunakan raw `db.Exec()` atau `db.Query()` dengan string concatenation
2. Dynamic filtering menggunakan `sqlc.narg()` (nullable args) — bukan string building
3. Untuk query yang tidak bisa di-handle sqlc (misal dynamic sort), gunakan whitelist kolom:

```go
// Aman — whitelist kolom yang diizinkan
var allowedSortColumns = map[string]bool{
    "full_name": true,
    "join_date": true,
    "department_id": true,
}

if !allowedSortColumns[sortBy] {
    sortBy = "full_name"  // default safe
}
```

4. Review kode secara berkala untuk memastikan tidak ada raw SQL queries

### 19.5 Security Headers

Semua response dari backend dan frontend WAJIB menyertakan header berikut:

| Header | Value | Fungsi |
|--------|-------|--------|
| `Strict-Transport-Security` | `max-age=63072000; includeSubDomains` | Paksa HTTPS (2 tahun) |
| `X-Content-Type-Options` | `nosniff` | Cegah MIME-type sniffing |
| `X-Frame-Options` | `DENY` | Cegah clickjacking |
| `Content-Security-Policy` | `default-src 'self'; ...` | Batasi sumber resource |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Batasi referer info |
| `Permissions-Policy` | `camera=(self), geolocation=(self)` | Batasi akses API browser |
| `Cross-Origin-Embedder-Policy` | `require-corp` | Keamanan cross-origin |
| `Cross-Origin-Opener-Policy` | `same-origin` | Isolasi origin |

**Implementasi Backend (Go Fiber):**

```go
app.Use(func(c *fiber.Ctx) error {
    c.Set("Strict-Transport-Security", "max-age=63072000; includeSubDomains")
    c.Set("X-Content-Type-Options", "nosniff")
    c.Set("X-Frame-Options", "DENY")
    c.Set("Referrer-Policy", "strict-origin-when-cross-origin")
    c.Set("Permissions-Policy", "camera=(self), geolocation=(self)")
    c.Set("Cross-Origin-Embedder-Policy", "require-corp")
    c.Set("Cross-Origin-Opener-Policy", "same-origin")
    return c.Next()
})
```

Jika backend dan frontend di-host terpisah, gunakan backend Fiber middleware untuk API response dan frontend SvelteKit hooks untuk HTML pages. Jangan konfigurasi di kedua tempat karena bisa menyebabkan konflik header.

```go
func (s *AuthService) Login(ctx context.Context, email, password string) (*LoginResponse, error) {
    // 1. Cek account lockout
    if emp.IsLocked && emp.LockedUntil.After(time.Now()) {
        return nil, ErrAccountLocked
    }

    // 2. Verify password
    if !CheckPassword(password, emp.PasswordHash) {
        // Catat failed attempt
        s.repo.RecordLoginAttempt(ctx, emp.ID, ip, false)

        // Cek total failed attempts dalam 15 menit
        failedCount := s.repo.CountRecentFailedAttempts(ctx, emp.ID)
        if failedCount >= 5 {
            s.repo.LockAccount(ctx, emp.ID, 15*time.Minute)
            return nil, ErrAccountLocked
        }
        return nil, ErrInvalidCredentials
    }

    // 3. Catat successful login + update last_login
    s.repo.RecordLoginAttempt(ctx, emp.ID, ip, true)
    s.repo.UnlockAccount(ctx, emp.ID)  // Reset lockout
    s.repo.UpdateLastLogin(ctx, emp.ID)

    // 4. Generate tokens
    accessToken := jwt.GenerateAccessToken(emp)
    refreshToken := jwt.GenerateRefreshToken()
    s.repo.CreateSession(ctx, emp.ID, refreshToken, deviceInfo, ip)

    return &LoginResponse{AccessToken: accessToken}, nil
}
```

**Refresh Token Rotation:**
- Setiap kali refresh token digunakan, session lama di-deactivate, buat session baru
- Jika refresh token yang sudah tidak aktif digunakan lagi → curiga session hijacking → semua session user di-deactivate + notifikasi

```go
// Refresh token rotation with hijacking detection
func (s *AuthService) Refresh(ctx context.Context, refreshToken string) (*RefreshResponse, error) {
    session, err := s.repo.GetSessionByRefreshToken(ctx, refreshToken)
    if err != nil {
        return nil, ErrInvalidToken
    }

    if !session.IsActive {
        // ⚠️ Token sudah dipakai sebelumnya — kemungkinan session hijacking!
        s.repo.DeactivateAllUserSessions(ctx, session.UserID)
        s.repo.NotifyUser(ctx, session.UserID, NotificationTypeSecurityAlert)
        return nil, ErrSessionHijacked
    }

    // Rotate: deactivate old, create new
    s.repo.DeactivateSession(ctx, session.ID)
    newRefreshToken := jwt.GenerateRefreshToken()
    s.repo.CreateSession(ctx, session.UserID, newRefreshToken, ...)

    return &RefreshResponse{
        AccessToken:  jwt.GenerateAccessTokenFromUserID(session.UserID),
        RefreshToken: newRefreshToken,
    }, nil
}
```

```go
        return ErrFileRequired
```
